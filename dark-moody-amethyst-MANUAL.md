# Dark Moody Amethyst — Manual Application Reference

Work through each category top to bottom. Every rule shows all properties in one place.

**Label Size:** The number in the top-right of each rule in FilterBlade. "default" = don't touch it.
**Icon Size:** Large = 0, Medium = 1, Small = 2 in FilterBlade dropdown.
**Beam:** All beams are permanent — do NOT check Temporary.
**Sound:** All volumes = 300.

## SAVE FILE FIX — Run Before Every Load

FilterBlade stores "off" states (disabled beams, transparent backgrounds, no sound) as `null` or `false` internally. When you download and re-upload a save file, these values crash FilterBlade on load with a `TypeError: Cannot read properties of undefined` error.

**Every time you download a save file, run the fix script before loading it back into FilterBlade.**

### How to run

1. Download the fix script: `fix_savefile.py`
2. Open a terminal and run:

```bash
python3 fix_savefile.py YourFilter.filterBladeSaveFile
```

This creates `YourFilter_fixed.filterBladeSaveFile` automatically. Load the fixed file into FilterBlade.

Or specify a custom output name:

```bash
python3 fix_savefile.py YourFilter.filterBladeSaveFile YourFilter_ready.filterBladeSaveFile
```

### What it fixes

- Removes `null` and `false` no-op entries for: `SetBackgroundColor`, `SetTextColor`, `SetBorderColor`, `PlayEffect`, `PlayAlertSound`, `MinimapIcon`, `UsedEntryStyleCombi`
- Converts any hex color values (`#C77DFF`) to rgb format (`rgb(199, 125, 255)`) which FilterBlade requires on load

### Why this happens

FilterBlade accepts hex colors and null values when you type them in the live editor, but its save/load parser is stricter and crashes on these values. The fix script normalizes the file to what the parser expects.
